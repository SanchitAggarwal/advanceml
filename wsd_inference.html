Background

What Is Word Sense Disambiguation?

Definition
Word Sense Disambiguation (WSD) is the task of determining which meaning of an ambiguous word is intended in a given sentence, using the surrounding words as evidence.

The Running Example: "bank"

Sense 1 — River
muddy river bank erodes quickly

Teal words = river-sense context clues

Sense 2 — Finance
loan approved bank charged interest

Blue words = finance-sense context clues

The challenge: both sentences have the same target word bank — but completely different meanings. A Bayesian Network can use the context words as evidence to infer which sense is intended.

Section 1

The Bayesian Network for WSD

Network Structure

The Sense of "bank" is the hidden root node. The context words in each position are the observable leaf nodes. Each context word's distribution depends on the sense.

Gold = Sense node (hidden). Teal = observed context words (w₁…w₅, excluding target position).

🔑 Key independence assumption: Given the Sense, each context word is conditionally independent of all other context words. This is the Naïve Bayes assumption — it makes inference tractable by allowing us to multiply per-word likelihoods.

Inference Formula

P(Sense | w₁,w₂,w₄,w₅) ∝ P(Sense) × ∏ᵢ P(wᵢ | Sense)

The prior P(Sense) reflects base rates (e.g. how often does "bank" mean river vs. finance in typical text). Each P(wᵢ | Sense) term is a row in a CPT that encodes how likely each word is under each sense.

Section 5

Inference Mechanics in WSD

⚡ Why WSD inference is tractable: The naive Bayes assumption (context words independent given Sense) means we never need to sum over exponentially many combinations. Each word contributes independently as a multiplicative factor.

Exact vs. Approximate in WSD

Exact (Naive Bayes)

With the independence assumption, the product over context words is computed exactly. O(|Senses| × |Words|). This is what the CPT approach does.

Without the assumption

We'd need P(w₁,w₂,w₄,w₅ | Sense) — a joint over all context words. With a vocabulary of 50,000 words and 4 positions, this table would be astronomically large.

08

Marginalizing the sense. Suppose instead of computing P(Sense | words), you wanted to compute P(w₃ = "erodes" | w₁ = "river", w₅ = "quickly") — the probability of a context word given two others, marginalizing out the Sense. Write this as a sum and explain each term.


P(erodes | river, quickly) = Σs ∈ {R,F} P(erodes | s) · P(s | river, quickly)

Term 1 (s=River): P(erodes|River) × P(River|river,quickly) = 0.12 × 0.9996 ≈ 0.1199
Term 2 (s=Finance): P(erodes|Finance) × P(Finance|river,quickly) = 0.001 × 0.0004 ≈ 0.0000004

Total ≈ 0.12. The Sense node acts as the bottleneck — to go from observed context words to a prediction about another context word, we must pass through the hidden Sense, which is the only parent of all context nodes. This is message passing through a hidden variable.

3. Updated formula:
P(Sense | w₁,…,w₄) ∝ P(Sense) × ∏ᵢ P(wᵢ | Sense)
where Sense ∈ {River, Finance, Storage} — the sum over three terms in normalization.

4. Prior update: We need P(River) + P(Finance) + P(Storage) = 1. If Storage is rarer in a news corpus, we might set P(Storage) = 0.05 and adjust: P(River) = 0.33, P(Finance) = 0.62. The constraint to sum to 1 is required because Sense is a proper probability distribution over mutually exclusive, exhaustive outcomes.
