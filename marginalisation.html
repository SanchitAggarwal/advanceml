Factorization of the Joint Distribution

The full joint distribution over all variables in a Bayesian Network can be written as a product of individual conditional probability tables (CPTs). This is called factorization — and it is the core algebraic identity that makes Bayesian Networks powerful.

P(JC, MC, A, E, B) = P(JC|A) · P(MC|A) · P(A|E,B) · P(E) · P(B)

Each factor corresponds to exactly one node's CPT — the probability of that node given only its direct parents in the graph. No approximation is involved; the factorized form is exactly equal to the full joint.

Key insight: The joint distribution of a Bayesian Network is the product of each node's conditional probability table given its parents. This is exact, not approximate.

Derivation — Chain Rule + Conditional Independence

Start: Apply the chain rule of probability

The chain rule is always valid — no assumptions about the network structure are required. We simply pick a valid ordering of variables and write the joint as a product of conditionals:

P(JC, MC, A, E, B) = P(JC | MC, A, E, B) × P(MC | A, E, B) × P(A | E, B) × P(E | B) × P(B)

Every term on the right conditions on all variables that appear before it in the chosen ordering. This expression is exact by definition.

Apply conditional independence from the graph structure

The Bayesian Network graph tells us that JohnCalls and MaryCalls depend only on Alarm. Once Alarm is observed, nothing else affects them:

P(JC | MC, A, E, B) → P(JC | A) ← only Alarm matters for JC P(MC | A, E, B) → P(MC | A) ← only Alarm matters for MC

Substituting these simplifications:

P(JC, MC, A, E, B) = P(JC | A) × P(MC | A) × P(A | E, B) × P(E | B) × P(B)

Simplify the root nodes

Earthquake and Burglary are root nodes — they have no parents in the graph. There is no causal path between them, so they are marginally independent:

P(E | B) → P(E) ← E and B are independent events

This is intuitive: whether a burglary happens does not influence whether an earthquake happens. They are separate causes with no shared parent.

The fully factorized joint distribution

Combining all simplifications from the previous steps:

P(JC, MC, A, E, B) = P(JC | A) · P(MC | A) · P(A | E,B) · P(E) · P(B)

In general, for any Bayesian Network with nodes X₁…Xₙ:

P(X₁, X₂, ..., Xₙ) = ∏ᵢ P(Xᵢ | Parents(Xᵢ))

Result: The full joint equals the product of all CPTs. Each node contributes exactly one factor, conditioned on its direct parents (or unconditionally if it is a root).

Conditional Probability Tables

Each node in a Bayesian Network has one CPT. For root nodes (no parents), it is an unconditional prior. For non-root nodes, each row corresponds to one configuration of the parent variables, and the row must sum to 1 — so one value per row is free.

Understanding CPTs via Contingency Tables

A contingency table (also called a cross-tabulation) shows the joint frequency or probability for combinations of two or more variables. In Bayesian Networks, CPTs are a form of conditional contingency table — they show P(child | parent combination).

Contingency table for P(Alarm | Burglary, Earthquake)

Rows = Alarm state, Columns = (Earthquake, Burglary) configuration. Each cell shows P(A | E, B). The 0.940 cell highlights the most common real-world scenario — burglary without earthquake.

Alarm \ (E, B) E=true, B=true E=true, B=false E=false, B=true E=false, B=false
A = true 0.950 0.290 0.940 0.001
A = false 0.050 0.710 0.060 0.999
Column sum 1.000 1.000 1.000 1.000

Important: Each column sums to 1.0. CPT rows are conditional distributions — given a fixed parent configuration, the probabilities across all child states must sum to 1. This is why a 4-row CPT has only 4 free parameters, not 8.

Contingency table for P(JohnCalls | Alarm) × P(Alarm | Burglary, Earthquake)

The last column is the marginalized probability P(JC=true | E, B) — obtained by summing across the A states (columns 1 and 3). This is exactly what marginalization computes, shown as table arithmetic.

4
Marginalization

When we want a probability but some intermediate variable is not observed, we use marginalization: sum over all possible values that unobserved variable could take. Each term in the sum is one "possible world."

P(X) = Σ_y P(X, Y=y) = Σ_y P(X | Y=y) · P(Y=y)

Rule: If a variable is not observed, sum over all its possible values. The total probability is the weighted average of the probability in each "world," weighted by the probability of that world.

The alarm example

We want to compute P(JohnCalls | Burglary, Earthquake). But JohnCalls depends on Alarm, and we don't observe Alarm. Alarm can be true or false, so:

P(JC | B, E) = P(JC | A=true) · P(A=true | B, E) + P(JC | A=false) · P(A=false | B, E)

The first term is the "alarm rang" world; the second is the "alarm silent" world. We weight each by its probability given the evidence.

General formula

P(JC=t | B, E) = 0.90 · P(A=t | B, E) ← alarm rang, John likely calls + 0.05 · P(A=f | B, E) ← alarm silent, John might still call

The values 0.90 and 0.05 come directly from the JohnCalls CPT. The alarm probability P(A | B, E) comes from the Alarm CPT. This is exactly why factorization and marginalization work together: the factorized CPTs are the lookup tables we use when marginalizing.

Connection: Marginalization uses the factorized CPTs as its building blocks. Factorization tells us how the joint splits; marginalization tells us how to handle unobserved variables in the chain.
