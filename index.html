What is a Bayesian Network?

A directed acyclic graph (DAG) where each node is a random variable and each edge represents a direct probabilistic influence. The graph encodes conditional independence assumptions — it tells us which variables are relevant to each other and which are not.

Conditional Probability Tables (CPTs)

Every node X stores a CPT: P(X | Par(X)). For a root node (no parents) this is just a prior P(X). The CPT tells you the probability of X for every combination of its parent values.

Joint Probability Factorisation

The full joint distribution factorises as a product over all nodes:

P(X₁,…,Xₙ) = ∏ P(Xᵢ | Par(Xᵢ))

This is what makes BNs compact — instead of storing 2ⁿ entries you store one small CPT per node.

Bayes' Theorem

The engine behind all inference in Bayesian Networks:

P(H|E) = P(E|H) · P(H) / P(E)

Posterior = Likelihood × Prior / Evidence. We update belief in H after observing E.

Local Semantics & Markov Blanket

Local semantics: Each node X is conditionally independent of all its non-descendants given its parents. This is the core independence assumption baked into the graph structure.

Markov Blanket of X = Parents ∪ Children ∪ Parents-of-Children. Given its Markov blanket, X is independent of every other node in the network.

MB(X) = Par(X) ∪ Child(X) ∪ Par(Child(X))

01 Chains (Serial Connections) Read First
What is a chain?

A chain is a linear path where information flows from one node to the next: A → B → C. The middle node B is neither a pure cause nor a pure effect — it sits in the middle, relaying influence from A to C.

Real example: Burglary → Alarm → MaryCalls. A burglary makes the alarm more likely; a ringing alarm makes Mary more likely to call. So burglary and Mary's call are related — not directly, but through the alarm.

The Chain Rule: In A → B → C, information flows freely when B is not observed. Conditioning on B blocks the path — A and C become conditionally independent given B.

A ⊥⊥ C | B    but    A ⊬⊥ C (marginally)
Why does observing B block the path?

Once you know whether the alarm rang, the burglary cannot give you any new information about Mary's call. The alarm state is the only thing Mary's call-probability depends on. There is no "extra channel" left for burglary to influence Mary.

01 Discuss Class Discussion
Q1 You know the alarm rang. Your friend tells you there was a burglary. Does this change how likely you think Mary called? Why or why not?
No. Once you know the alarm rang, you already have all the information that determines Mary's calling probability — specifically P(Mary | Alarm = true). Whether a burglary caused that alarm is irrelevant to Mary's decision to call; she only responds to the alarm itself. The burglary is "screened off" by the alarm.
Q2 In a chain A → B → C → D, if you observe B, does that block information flow between A and D?
Yes. Observing B blocks the path at B. Since the only path from A to D passes through B, and B is observed and is a chain node, the entire path is blocked — regardless of what happens at C or D. A ⊥⊥ D | B. In general, one blocked node on a path is enough to block the whole path.

02 Forks (Common Cause) Read First
What is a fork?

A fork has a single parent node with two or more children: A ← B → C. The parent is the common cause. The children are not directly connected to each other — but they appear correlated because they share the same parent.

Real example: JohnCalls ← Alarm → MaryCalls. The alarm causes both calls. Without knowing the alarm state, observing John called gives you information about Mary calling — they look correlated.

The Fork Rule: In A ← B → C, information flows freely when B is not observed. Conditioning on B blocks the path — the children A and C become conditionally independent given B.

A ⊥⊥ C | B    but    A ⊬⊥ C (marginally)
Intuition — the "explaining parent" idea

Once you know the alarm state, it fully explains both John's and Mary's calling behaviour. John calling gives you zero additional information about Mary's call — the common cause has already been "accounted for." The parent screens off the children from each other.

Analogy (Berkson's paradox): Two symptoms (fever, rash) both caused by a disease. If you know the disease is present, learning about fever tells you nothing new about rash — the disease already explains both. But if you don't know the disease status, fever and rash look correlated.

02 Discuss Class Discussion
Q3 Two students both score well on an exam. Is this a coincidence, or are their scores related? What's the "fork" structure here?
Their scores are related through a common cause — the exam difficulty, quality of teaching, or shared preparation material. The fork is: StudentA_Score ← ExamDifficulty → StudentB_Score. Without knowing the exam difficulty, high scores appear correlated. Once you control for difficulty (observe it), the scores become independent — one student's score tells you nothing extra about the other's.
Q4 In a fork, can the two children ever influence each other even after observing the parent? When might this happen in a real Bayesian network?
In a pure fork with only this one path, no — observing the parent fully blocks the path. However, in a real network there may be other paths connecting the two children (e.g., a direct edge between them, or another common ancestor). In that case, blocking the fork path is not enough — you'd need all paths blocked to achieve independence. This is exactly what d-separation checks.

03 Colliders & d-Separation Read First
What is a collider?

A collider has two or more parents pointing into a single child: A → B ← C. The child B is where the arrows "collide." Unlike chains and forks, the two causes A and C are independent by default — but observing the effect B (or any descendant of B) makes them dependent.

Real example: Earthquake → Alarm ← Burglary. Earthquakes and burglaries happen for completely different reasons. But once you know the alarm rang, the two causes start influencing each other's likelihood.

Explaining Away

This phenomenon has a name: explaining away. Once you know the alarm rang and you learn a burglary occurred, the burglary explains away the need for an earthquake — so your belief in an earthquake decreases. Conversely, if you rule out a burglary, your belief in an earthquake increases to explain the alarm.

The two causes compete to explain the single observed effect.

The Collider Rule (OPPOSITE to chain/fork): In A → B ← C, the path is blocked by default. Conditioning on B (or any descendant of B) opens the path — A and C become dependent.

A ⊥⊥ C    but    A ⊬⊥ C | B (and A ⊬⊥ C | descendant of B)
d-Separation — putting it all together

d-separation is the graphical algorithm that tests conditional independence between any two nodes X and Y given an evidence set Z. The idea: list every undirected path between X and Y, then check whether each path is blocked or open according to the three rules. If ALL paths are blocked → X and Y are d-separated → conditionally independent given Z.

StructureWithout observing middle/childObserving middle/child
Chain A→B→C PATH OPEN A,C dependent PATH BLOCKED A ⊥⊥ C | B
Fork A←B→C PATH OPEN A,C dependent PATH BLOCKED A ⊥⊥ C | B
Collider A→B←C PATH BLOCKED A ⊥⊥ C PATH OPENS A ⊬⊥ C | B
d-Separation algorithm (step by step)

Step 1: List all undirected paths between X and Y (ignore edge directions when finding paths, just follow connections).

Step 2: For each path, examine every node on that path. Classify each node as chain, fork, or collider with respect to that path (same node can be different types on different paths).

Step 3: A path is blocked if any node on it satisfies: (a) it's a chain/fork middle node and it IS in Z, or (b) it's a collider and neither it nor any of its descendants is in Z.

Step 4: If ALL paths are blocked → d-separated → X ⊥⊥ Y | Z. If any path is open → dependent.

03 Discuss Class Discussion
Q5 You apply for a competitive scholarship. The selection committee picks candidates who are either talented OR from an under-represented background (or both). You get selected. Now, does knowing you're from an under-represented background affect beliefs about your talent? What structure is this?
This is a collider — Talent → Selected ← Background. By default, Talent and Background are independent. But once you condition on "Selected = yes" (the collider), they become negatively correlated: knowing you're from an under-represented background slightly lowers the posterior belief that raw talent was needed to get selected. This is Berkson's paradox and it's a collider in action. It's also why studies that select on a collider produce misleading correlations.
Q6 What's the key difference in how chain/fork vs collider respond to conditioning? Why is the collider "backwards"?
In chains and forks, the middle/parent node acts as the intermediary or common cause. Observing it gives you all the information connecting the two endpoints — so they become independent.

In a collider, the child node is the common effect. By default there's no connection. But observing the effect creates a "competition" between the two causes — if one cause is confirmed, the other becomes less likely, and vice versa. Conditioning creates a dependency that wasn't there before. It's backwards because information flows up from the effect to the causes.

Path: A → B → ? — wait, there's no direct B→F edge. The only path from A to F is:
A → B ← C → F

Node roles on this path:
• B (A→B←C): collider. B is in Z (observed). → This opens the collider at B.

EX 07 Exam-style: Find the evidence set HARD

Network: R → S → W ← T (R=Rain, S=Sprinkler, W=WetGrass, T=Thunder)

Find an evidence set Z such that R ⊥⊥ T | Z. There may be more than one valid answer. Justify your reasoning.

The only path from R to T is: R → S → W ← T

This path has two nodes to check:
• S (R→S→W): a chain middle node. If S ∈ Z → blocks here.
• W (S→W←T): a collider. If W ∉ Z and no descendant of W ∈ Z → blocks here.

Valid Z sets:
1. Z = ∅ (empty): W is a collider, not observed, no descendant observed → path blocked at W. R ⊥⊥ T. ✓
2. Z = {S}: S is a chain node and is observed → path blocked at S. R ⊥⊥ T | S. ✓

Invalid Z sets (make them DEPENDENT):
• Z = {W}: W is a collider, now observed → path opens. R ⊬⊥ T | W. ✗

Key insight: for this path, you can achieve independence either by blocking the chain (observe S) or by leaving the collider alone (don't observe W).

Full network: A → B → D → E ← F and C → B. Query: Is A ⊥⊥ F given {B, E}?

Work through every path systematically using the table below.

Step 1 — list all paths from A to F
Step 2 — for each path, is it blocked or open? Why?
Step 3 — conclusion
Step 1 — Paths from A to F:
Only one path exists: A → B → D → E ← F

Step 2 — Check nodes on this path with Z = {B, E}:
• B (A→B→D): chain/fork middle node. B ∈ Z (observed) → BLOCKS here.
• D (B→D→E): chain middle node. D ∉ Z → open at D.
• E (D→E←F): collider. E ∈ Z (observed) → OPENS at E.

One node blocks (B), one opens (E). The path is blocked — a single blocked node is enough to block the whole path.

Step 3 — Conclusion: A ⊥⊥ F | {B, E} ✓

Note: Even though observing E (the collider) tries to open the path, B blocks it upstream first. The path only needs to be blocked at one point.
