Hidden Markov Model — Forward Algorithm

Problem Statement

We want to predict the next word in a partially written sentence or email using an HMM. The core insight: hidden states represent grammatical/semantic context (e.g. topic, POS tag), while observations are the actual words the user types.

Two Use Cases

Use Case A Email writing assistant — predict the next word given subject & partial body (e.g. "Please find attached the…")

Use Case B Sentence completion — general next-word prediction for any context (e.g. "The error in the data…")

Why HMM?

Language has latent structure. The word "report" after "please find attached the" is likely because we are in a FORMAL_EMAIL state — a state we cannot directly observe but can infer probabilistically via the Forward Algorithm. Unlike n-grams, HMMs explicitly model this hidden context.

HMM = ( S, V, A, B, π )
S = hidden states e.g. {FORMAL_EMAIL, CASUAL_EMAIL, TECHNICAL, CLOSING}
V = vocabulary of observed words
A = transition matrix A[i][j] = P(state_j | state_i)
B = emission matrix B[s][w] = P(word_w | state_s)
π = initial state distribution π[s] = P(state_s at t=1)
markdown 🔬 Approach — Forward Algorithm Step-by-Step

The Forward Algorithm

The forward variable α_t(s) = probability of having observed all words up to time t AND being in state s at time t. It is computed via dynamic programming (left-to-right recursion).

① Initialization (t = 1):
α₁(s) = π(s) · B(s, w₁) for all states s

② Recursion (t = 2 … T):
α_t(s) = [ Σ_{s'} α_{t-1}(s') · A(s' → s) ] · B(s, w_t)

③ Next-word scoring (after seeing T words):
score(w_next) = Σ_s [ Σ_{s'} α_T(s') · A(s' → s) ] · B(s, w_next)

④ Return top-k words by score → predictions
💡 Normalization at each step prevents floating-point underflow for long sequences. The normalized α still gives correct relative rankings for next-word prediction.

Key Insight

At the end of the observed sequence, α_T encodes a belief distribution over which hidden state we are in. We then project forward through A and B to score every candidate next word — no Viterbi path needed, just a weighted sum.

code [3] ⚡ Forward Algorithm — Core Implementation 
def forward(obs_words, pi, A, B, vocab):
    """
    Run the HMM Forward Algorithm.
    Args:
        obs_words : list[str]  — observed word sequence
        pi        : np.array   — initial state distribution
        A         : np.array   — transition matrix (n_states × n_states)
        B         : np.array   — emission matrix   (n_states × vocab_size)
        vocab     : dict       — word → index mapping
    Returns:
        alpha     : np.array   — (T × n_states) forward variables
    """
    obs = [vocab[w] for w in obs_words if w in vocab]
    T, n = len(obs), len(pi)
    alpha = np.zeros((T, n))

    # ── Step ①: Initialization ─────────────────────────────────────────
    alpha[0] = pi * B[:, obs[0]]
    alpha[0] /= alpha[0].sum() + 1e-12        # normalize for stability

    # ── Step ②: Recursion ──────────────────────────────────────────────
    for t in range(1, T):
        for s in range(n):
            # sum over all previous states weighted by transition
            alpha[t, s] = np.dot(alpha[t-1], A[:, s]) * B[s, obs[t]]
        alpha[t] /= alpha[t].sum() + 1e-12    # scale to prevent underflow

    return alpha   # shape: (T, n_states)


def predict_next(obs_words, pi, A, B, vocab, top_k=5):
    """
    Score all vocabulary words as next-word candidates using α_T.
    Returns sorted list of (word, score) tuples.
    """
    idx_to_word = {v: k for k, v in vocab.items()}
    alpha = forward(obs_words, pi, A, B, vocab)
    alpha_T = alpha[-1]      # final state belief distribution

    scores = {}
    for w_idx, w in idx_to_word.items():
        score = 0.0
        for s in range(len(pi)):
            # probability of transitioning into state s from current belief
            p_next_state = np.dot(alpha_T, A[:, s])
            score += p_next_state * B[s, w_idx]
        scores[w] = score

    return sorted(scores.items(), key=lambda x: x[1], reverse=True)[:top_k]

markdown 📝 Summary & Extensions

What We Built

A complete HMM-based next-word predictor that uses the Forward Algorithm to maintain a belief distribution over hidden states (email context) and uses it to score candidate next words. The model correctly predicts "report" after "please find attached the" and "data" after "function error".

Extensions

Scale up: Replace the toy vocabulary with a real corpus. Train A and B using the Baum-Welch (EM) algorithm rather than hand-crafting them.

Higher-order HMM: Condition transitions on the last k states (increases expressivity at the cost of exponential state space growth).

Hybrid approach: Use HMM hidden states as context features fed into a neural language model — combining interpretable latent structure with deep learning expressiveness.

Time complexity of Forward pass : O(T × N²) where T = sequence length, N = states
Space complexity : O(T × N) (can be reduced to O(N) if only final α_T needed)
