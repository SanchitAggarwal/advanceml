Section 1

What is Inference?

Definition
Inference in a Bayesian Network means computing the posterior probability of some query variables given observed values of evidence variables.
P(Query | Evidence) = ?
💡 The key challenge: variables that are not observed must be summed out (marginalized). This is what makes inference non-trivial — and potentially expensive.

02

Marginalization. Suppose Alarm (A) is not observed. Write the sum that marginalizes A out of the expression P(B) · P(A|B) · P(JC|A):

P(B, JC) = Σa P(B) · P(a | B) · P(JC | a)

What does each term in the sum represent? Write your answer below.

For a = true: We assume the alarm rang — then we weight by how likely B causes the alarm to ring, and how likely John calls when the alarm is on.

For a = false: We assume the alarm didn't ring — same logic but with those (lower) probabilities.

Adding both terms gives the total probability of (B, JC), regardless of what Alarm actually was. This is marginalization.

Definition
Message passing is a technique where each node computes a local summary (a "message") and passes it to its neighbours. Beliefs are updated by combining incoming messages — no node needs to see the entire network.

Forward Sampling

Definition
Forward sampling generates a complete assignment of all variables by sampling each node in topological order (parents before children), using that node's conditional probability table given its already-sampled parents.
📋 Topological order for the Burglar Alarm network: B → E → A → JC → MC. Sampling must respect this order.

06

Rejection sampling efficiency. Using the simulator above, set the filter to "JC=true AND MC=true" and run 200 samples. Observe how many samples are kept versus discarded. Then explain why this is inefficient when the evidence is rare.

When evidence is rare (e.g. P(JC=T, MC=T) is small because it requires a burglary or earthquake first), most generated samples won't match the filter and are discarded. Even with 200 samples, you might keep only a handful — making your frequency estimates very noisy. This is the fundamental inefficiency of rejection sampling: computational cost is inversely proportional to the probability of the evidence.
Section 4

Exact vs Approximate Inference

Exact Inference

Computes the true posterior. Works by manipulating CPTs directly — multiplying and marginalizing.

Cost: Exponential in the number of variables in the worst case (NP-hard for general networks).

Approximate Inference

Trades accuracy for speed. Generates random samples and estimates probabilities from frequencies.

Cost: Scales with the number of samples needed for accuracy — often far more tractable.

Section 5

Limitations of Bayesian Networks

Five key limitations
  1. Inference can be computationally expensive (NP-hard in general)
  2. CPTs become exponentially large as the number of parents grows
  3. Learning the graph structure from data is hard (often requires expert knowledge)
  4. Cycles are forbidden — feedback loops require extensions like Dynamic BNs
  5. Wrong independence assumptions lead to systematically wrong posteriors

Putting It All Together

Open-ended: Design your own network

Consider the following scenario: A student is late to class. This could be because of (a) a train delay, or (b) oversleeping. A train delay also depends on whether there was a track maintenance on that day.

Answer the following in the space below:

  1. Draw (describe in text) the Bayesian Network structure for this scenario.
  2. List all the CPTs you would need to specify.
  3. Write the inference query for: "Given the student is late, what is the probability of a track maintenance?"
  4. Identify any independence assumptions encoded in your graph and assess whether they are realistic.
Network structure:
Maintenance → TrainDelay → Late ← Oversleeping
(Maintenance is the sole parent of TrainDelay; both TrainDelay and Oversleeping are parents of Late)

CPTs needed:
• P(Maintenance) — prior on maintenance days
• P(TrainDelay | Maintenance) — conditional on maintenance
• P(Oversleeping) — prior, independent root
• P(Late | TrainDelay, Oversleeping) — 4-row table (2 binary parents)

Inference query:
P(Maintenance | Late=T) = α · Σ_{td, os} P(Maintenance) · P(td | Maintenance) · P(os) · P(Late=T | td, os)

Independence assumptions:
• Oversleeping ⊥ Maintenance (plausible — sleeping habits independent of city events)
• Oversleeping ⊥ TrainDelay | nothing (plausible)
• The only path from Maintenance to Late is through TrainDelay (encoded by the graph)
Potential violation: students may set early alarms specifically because they worry about train delays — creating a dependence between Oversleeping and Maintenance not captured in the graph.
