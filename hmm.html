01 Why We Need HMMs — The Story
🆕 The Setup

Imagine you're a professor. You look out at the class. You can't read minds — but you need to figure out what's actually going on inside each student's head. All you can see is what they do, not what they think.

🧠 What's in the Student's Mind

The true state — nobody can see this directly

Focused (F) Distracted (D) Zoned Out (Z)
👁 What the Professor Sees

The observations — visible behaviour signals

Typing notes (T) Scrolling phone (S) Nodding head (N)
HMM Structure — Two Layers
🔑 The Core Insight

The professor can only see the bottom row (observations). The top row (mental states) is completely hidden. The dashed arrows show that each hidden state emits an observation with some probability — it's not a perfect signal.

A Focused student still scrolls sometimes. A Distracted student still types sometimes. That's what makes this hard — and real.

02 The Five Components of an HMM

An HMM is completely described by λ = (A, B, π). Here's every piece, with the actual numbers we'll use throughout this guide.

Interactive — click any component to see its explanation
Think of it like this

The hidden states are the real story happening inside the student's mind. Like chapters of a book no one else can read.

Q = {F (Focused), D (Distracted), Z (Zoned Out)}
N = 3 hidden states total
At every time step, the student is in exactly ONE of these states.
Think of it like this

Observations are the clues the professor can see — but they don't perfectly reveal the truth. Like seeing footprints: you know someone walked there, but not why.

V = {T (Typing), S (Scrolling), N (Nodding)}
M = 3 possible observations
At every time step, exactly ONE observation is produced.
Think of it like this

Transitions show how the mind moves from one state to another. A Focused student usually stays focused — but sometimes drifts. Just like weather: if today is sunny, tomorrow is probably also sunny, but not always.

A[i][j] = P(go to state j | currently in state i)

            → F    D    Z
From F  [0.7   0.2   0.1]
From D  [0.3   0.5   0.2]
From Z  [0.1   0.3   0.6]
Each row sums to 1.0 (you must go somewhere!)
Notice

The diagonal values (0.7, 0.5, 0.6) are the highest in each row. This means students tend to stay in their current state. Once focused, you're more likely to stay focused. Once zoned out... you usually stay there.

Think of it like this

Emissions show how each hidden state produces observable behaviour. This is the "noise layer." A Focused student mostly types, but scrolls sometimes. A Distracted student mostly scrolls, but types occasionally too.

B[i][k] = P(observe k | currently in state i)

             → Type  Scroll Nod
State F  [0.6    0.1    0.3]
State D  [0.1    0.7    0.2]
State Z  [0.2    0.4    0.4]
Each row sums to 1.0
Key Insight

If emissions were an identity matrix (1 on diagonal, 0 elsewhere), then each state would always produce one specific observation — and we'd just have a regular Markov Chain. The "blurriness" of B is exactly what makes it hidden.

Think of it like this

The initial distribution is like asking: when the professor first looks up at the class at t=0, what are the chances each student is Focused, Distracted, or Zoned Out? (Before anything has been observed.)

π = [0.6, 0.3, 0.1]
= [P(start Focused), P(start Distracted), P(start Zoned Out)]
Sums to 1.0 — you have to start somewhere!
⚡ Complete HMM in One Line

λ = (A, B, π) — That's it. Give me a transition matrix, an emission matrix, and a starting distribution, and I've fully defined an HMM. Everything else (the three algorithms) are ways to use this model.

03 The Three Fundamental Problems

Every time you work with an HMM, you're trying to solve one of three problems. Think of them as three questions a professor might ask:

🎯
Evaluation

"Given what I saw, how likely is this model?"

P(O | λ)

Solved by: Forward Algorithm

🔎
Decoding

"What hidden states best explain what I saw?"

argmax P(Q|O,λ)

Solved by: Viterbi Algorithm

🏫
Learning

"How do I find the best A, B, π from data?"

argmax P(O | λ)

Solved by: Baum-Welch (EM)

04 The Forward Algorithm
🤔 The Problem We're Solving

The professor observes: Typing → Scrolling → Nodding. Question: How likely is this exact observation sequence, given our HMM?

Formally: compute P(O | λ) where O = (T, S, N).

Why brute force fails

The naive approach: enumerate every possible hidden state sequence, compute the probability each one generates O, and add them up.

💡 The Key Insight

Two paths that share the same first k steps are doing identical work when we compute them separately. Instead: compute partial results once, store them, and reuse. This turns exponential into polynomial.

Brute Force: O(N^T × T)    ← exponential, impossible
Forward Algo: O(N² × T)      ← polynomial, fast!
For T=100, N=3: 10⁴⁷ operations vs just 900

The alpha table — what we compute

We define a quantity α_t(i), called the forward variable:

α_t(i) = P(o₁, o₂, ..., o_t, AND q_t = i | λ)

Plain English: "The probability of having seen observations o₁...o_t so far,
AND being in hidden state i at time t."

Think of it as a table — one row per hidden state, one column per time step. We fill it left to right. Each cell answers: "What's the total probability of all paths that end up here?"

Forward Algorithm — Formula Reference
Step 1 — Initialization (t = 1)
α₁(i) = π(i) × B(i, o₁)

Step 2 — Recursion (t = 2, 3, ..., T)
α_{t+1}(j) = [ Σᵢ α_t(i) × A(i→j) ] × B(j, o_{t+1})

Step 3 — Termination
P(O | λ) = Σᵢ α_T(i)

Complexity: O(N²T) — polynomial!
05 The Viterbi Algorithm
🤔 The Problem We're Solving

The professor saw: Typing → Scrolling → Nodding. Now the question is: What was the student most likely actually thinking at each step?

Formally: find the hidden state sequence Q* that maximises P(Q | O, λ).

💡 Forward vs Viterbi — What's the Difference?

Forward Algorithm
Uses SUM over all paths.
Answers: "Total probability of observations"

Viterbi Algorithm
Uses MAX over all paths.
Answers: "Best single hidden path"

Same structure. One tiny change: replace Σ (sum) with max. That's it.

Viterbi uses a similar table, but instead of storing "sum of all paths," each cell stores "probability of the best path ending here." We also remember which path got us here, so we can trace back the answer.

Define δ_t(i) = probability of the BEST path that ends in state i at time t

Step 1 — Initialization
δ₁(i) = π(i) × B(i, o₁)

Step 2 — Recursion
δ_{t+1}(j) = MAX_i [ δ_t(i) × A(i→j) ] × B(j, o_{t+1})
ψ_{t+1}(j) = argmax_i [ δ_t(i) × A(i→j) ] ← remember where we came from

Step 3 — Backtrack
q*_T = argmax_i δ_T(i)
q*_t = ψ_{t+1}(q*_{t+1}) ← follow the backpointers

06 Baum-Welch — Learning the Model
🤔 The Problem We're Solving

What if we don't know A, B, and π? What if we're handed a pile of observation sequences, and we need to figure out the model ourselves?

Example: A new professor arrives. They've never taught before. They observe students for a week (lots of Typing, Scrolling, Nodding sequences). From this data alone, can they learn what mental states exist and how students transition between them?

Yes — using the Baum-Welch Algorithm, which is a special case of the famous Expectation-Maximization (EM) framework. It's like calibrating your model from evidence.

💡 The Big Idea — A Chicken-and-Egg Problem

To estimate A and B, we need to know the hidden states. But to know the hidden states, we need A and B. This is a chicken-and-egg problem!

Baum-Welch breaks the deadlock: guess A, B, π → use them to estimate hidden states → use those to improve A, B, π → repeat until stable.

Baum-Welch — The EM Loop
The Two Steps, Explained Simply
E-Step
Expectation — "What probably happened?"

Using the current (possibly wrong) model, compute a soft estimate of which hidden states were visited and which transitions were taken. We use two quantities:

γ_t(i) = P(q_t = i | O, λ) ← prob of being in state i at time t
ξ_t(i,j) = P(q_t = i, q_{t+1} = j | O, λ) ← prob of transition i→j at time t

Both computed using Forward (α) AND Backward (β) variables
Why "Backward" too?

The forward variable tells you "probability of everything before time t." The backward variable β_t(i) tells you "probability of everything after time t." Together, α × β gives you "probability of everything, given you're in state i at time t." That's γ.

γ_t(i) = (α_t(i) × β_t(i)) / P(O | λ)
M-Step
Maximization — "Update the model"

Use the soft estimates to re-estimate A, B, and π. Think of it as: "Given we think the student was probably Focused 70% of the time they typed, let's update B accordingly."

π̂(i) = γ₁(i) ← expected state at t=1

Â(i→j) = Σ_t ξ_t(i,j) / Σ_t γ_t(i) ← expected transitions i→j / expected time in i

B̂(i,k) = Σ_{t:o_t=k} γ_t(i) / Σ_t γ_t(i) ← expected times in state i AND observing k

⚠ Important Caveat

Baum-Welch finds a local maximum, not necessarily the global one. Different random initializations can lead to different final models. In practice, run multiple times with different starting points and pick the best.

Also, it can be slow to converge. Modern alternatives (like neural network-based sequence models) are often preferred for large datasets.

🎉 The One Thing To Remember

The world is full of problems where the true state is hidden and observations are noisy signals. Markov Chains assume you can always see the state. HMMs accept that you can't — and give you three powerful tools to reason about what's really going on beneath the surface.

Forward = "How likely is what I saw?" (SUM)
Viterbi = "What most likely happened?" (MAX)
Baum-Welch = "Learn the model from data" (EM)
